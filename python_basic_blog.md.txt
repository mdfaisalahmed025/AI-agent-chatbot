## Getting Started with Python

Python is known for its readability and simplicity, making it an ideal first language. To start writing scripts, you’ll need:

1. **Python interpreter** – Download the latest version from the official website (python.org) or use a package manager like `brew` or `apt`.
2. **A code editor** – VS Code, PyCharm, Sublime Text, or even a simple text editor like Notepad++ work well.
3. **Command‑line basics** – Learn how to navigate directories and run `python script.py`.
4. **Virtual environment tool** – `venv` or `conda` to keep dependencies isolated.

### Your First Script
```python
# hello_world.py
print("Hello, world!")
```
Run it with:
```
python hello_world.py
```
You should see `Hello, world!` printed to the console.

## Sample Scripts for Everyday Automation

### 1. File Organizer
```python
import os, shutil

source = "./Downloads"
for filename in os.listdir(source):
    if filename.lower().endswith('.pdf'):
        shutil.move(os.path.join(source, filename), './PDFs')
    elif filename.lower().endswith(('.jpg', '.jpeg', '.png')):
        shutil.move(os.path.join(source, filename), './Images')
    elif filename.lower().endswith('.txt'):
        shutil.move(os.path.join(source, filename), './Text')
```
This script sorts PDFs, images, and text files into dedicated folders.

### 2. Simple Web Scraper
```python
import requests
from bs4 import BeautifulSoup

url = "https://news.ycombinator.com/"
response = requests.get(url, timeout=10)
response.raise_for_status()

soup = BeautifulSoup(response.text, "html.parser")
for title in soup.select('.storylink'):
    print(title.get_text())
```
Grab the latest headlines from Hacker News with just a few lines and proper error handling.

### 3. CSV Data Cleaner
```python
import pandas as pd

# Load raw data
df = pd.read_csv('raw_data.csv')

# Basic cleaning steps
df_clean = (
    df.dropna(subset=['id', 'value'])          # Remove rows missing critical columns
      .assign(value=lambda x: x['value'].astype(float))  # Ensure numeric type
      .query('value >= 0')                     # Filter out negative values
)

# Save cleaned data
df_clean.to_csv('clean_data.csv', index=False)
```
A quick way to sanitize data before analysis.

## Tips for Writing Clean Python Code

1. **Follow PEP 8** – Consistent style makes your code readable.
2. **Use meaningful variable names** – `total_price` is clearer than `tp`.
3. **Leverage list comprehensions** – They’re concise and often faster.
4. **Document with docstrings** – Explain what functions do and their parameters.
5. **Write tests** – Even simple `assert` statements catch bugs early.
6. **Format with Black** – Run `black .` to automatically format your code.
7. **Static analysis** – Tools like `flake8` or `pylint` catch common mistakes.

## Debugging and Testing

- **Built‑in debugger** – Insert `import pdb; pdb.set_trace()` or run `python -m pdb script.py`.
- **Logging** – Use the `logging` module instead of `print` for production‑ready diagnostics.
- **Pytest fixtures** – Reuse setup code across multiple tests.
- **Coverage** – `pytest --cov=your_module` shows which lines are exercised by tests.

Example test with pytest:
```python
def add(a, b):
    return a + b

def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
```
Run with `pytest test_my_module.py`.

## Virtual Environments and Dependency Management

Keeping project dependencies isolated prevents version conflicts:
```bash
# Create a virtual environment
python -m venv .venv
# Activate (Linux/macOS)
source .venv/bin/activate
# Activate (Windows)
.venv\Scripts\activate
# Install requirements
pip install -r requirements.txt
```
A typical `requirements.txt` might look like:
```
requests==2.31.0
beautifulsoup4==4.12.2
pandas==2.2.0
click==8.1.7
pytest==8.0.0
black==24.1.0
```

## Packaging Your Scripts

When your script grows, consider turning it into a package:
```bash
mkdir mytool
cd mytool
mkdir mytool
touch mytool/__init__.py
```
Add an entry point in `setup.cfg`:
```ini
[options.entry_points]
console_scripts =
    mytool = mytool.__main__:main
```
Now users can install it with `pip install .` and run `mytool` from the command line.
